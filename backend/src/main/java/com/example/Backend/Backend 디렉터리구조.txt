
---------------------------------------------------------------------------------------------------

config
→  역할: 애플리케이션 전역 설정.

   내용: CORS, WebMvc 설정, Jackson, Swagger/OpenAPI, JPA, Cache, MessageConverter, AOP, 빈 등록.

   애너테이션: @Configuration, 필요 시 @Enable...

   주의: 비즈니스 로직 X.

---------------------------------------------------------------------------------------------------

controller
 →  역할: HTTP 엔드포인트. 입력 검증 → 서비스 호출 → DTO 응답.

    내용: @RestController, 요청 DTO/응답 DTO 사용, 예외는 전역 처리에 위임.

    애너테이션: @RestController, @RequestMapping, @Validated

    의존: service 만 참조 (entity/repository 직접 접근 금지).

---------------------------------------------------------------------------------------------------

dto
 →  역할: 입출력 전용 데이터 구조. (엔티티와 분리)

    내용: Request/Response DTO, 페이징 응답 래퍼 등.

    애너테이션: jakarta.validation(@NotNull, @Size…), record 사용 권장.

---------------------------------------------------------------------------------------------------

entity
 →  역할: 영속 도메인 모델(JPA 엔티티).

    내용: 필드, 연관관계, 불변/도메인 메서드, 낙관적 락 등.

    애너테이션: @Entity, @Table, @Id, @GeneratedValue, @Column

     주의: API에 엔티티 직노출 금지(반드시 DTO로 변환). 양방향 연관관계 과용 금지.

---------------------------------------------------------------------------------------------------

service
 →  역할: 비즈니스 규칙/트랜잭션 경계.

    내용: 도메인 규칙 구현, 트랜잭션 관리, Repository 호출, Entity↔DTO 매핑 위임.

    애너테이션: @Service, @Transactional

    의존: repository, mapper, (필요 시) 다른 service. controller는 모름.

---------------------------------------------------------------------------------------------------

repository
 →  역할: 데이터 접근 계층(DB).

    내용: Spring Data JPA 인터페이스, Querydsl/명시 쿼리.

    애너테이션: @Repository(인터페이스는 생략 가능), @Query

    의존: JPA 엔티티만. 서비스 외 레이어 모름.

---------------------------------------------------------------------------------------------------

exception
 →  역할: 공통 예외, 전역 예외 처리.

    내용: 커스텀 예외, @ControllerAdvice로 에러 응답 표준화.

---------------------------------------------------------------------------------------------------

mapper
 →  역할: Entity ↔ DTO 변환을 한 곳에 모음(중복/누수 방지).

    내용: MapStruct 권장(컴파일 타임), 아니면 수동 매핑.

    애너테이션: @Mapper(componentModel="spring")

---------------------------------------------------------------------------------------------------

security
 →  역할: 인증/인가, 필터, 보안 설정.

    내용: SecurityFilterChain, JWT, UserDetailsService, PasswordEncoder.

    애너테이션: @Configuration, @EnableMethodSecurity

---------------------------------------------------------------------------------------------------
